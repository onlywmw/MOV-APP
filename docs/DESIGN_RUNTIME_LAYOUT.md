# DESIGN: 运行页布局重构 — 信息层级

版本: v1.0
日期: 2026-07-22
状态: 📐 design-ready

---

## 问题

运行页现在从上到下有 7 个区块平铺——进程、通道、Cron、模型、权限、技能。用户打开运行页要滚动很久才能看完。其中通道、权限、技能是**低频信息**——用户不会每次都看。它们占据了高频信息（AI 模型状态、Cron 任务）的屏幕空间。

---

## 方案：三层信息架构

```
┌─ 运行 ───────────────────────────────────┐
│                                          │
│  ┌─ 第一层：核心 ───────────────────────┐ │
│  │ 进程卡片 (mov-agent · 运行状态)       │ │
│  │ AI 模型列表 (DeepSeek · Claude · ...) │ │
│  │ Cron 任务列表                         │ │
│  └──────────────────────────────────────┘ │
│                                          │
│  ┌─ 第二层：辅助 · 可折叠 ──────────────┐ │
│  │ ▶ 通道 · 应用壳/AI网关/小组件/通知   │ │
│  │ ▶ 技能 · 3 个                         │ │
│  │ ▶ 权限 · 相机/定位/联系人/短信/电话   │ │
│  └──────────────────────────────────────┘ │
│                                          │
└──────────────────────────────────────────┘
```

**第一层永远展开。** 进程、模型、Cron——用户打开运行 tab 就是要看这些。

**第二层默认折叠。** 每条是一行摘要。点 ▶ 展开看完整内容。用户不需要的时候不占地方。

## 第二层：折叠行

每行一条摘要。折叠时只占 40px。展开时显示完整内容。箭头旋转表示状态。

### 折叠态（默认）

```
▶ 通道 · 🟢🟢🟢🟢  全部正常
▶ 技能 · 3 个
▶ 权限 · 4/7 已授权
```

- 一行文字说清状态。用户不需要打开就知道有没有问题。
- 通道 4 个全绿 → "全部正常"。有一个红 → "AI 网关异常"。
- 权限 7 个授权了 4 个 → "4/7 已授权"。全部授权 → 不显示这行（自动隐藏）。

### 展开态（点 ▶ 或整行）

```
▼ 通道 · 全部正常
  ┌── 应用壳 · 🟢 ──┬── AI 网关 · 🟢 ──┐
  │ 小组件 ×2 · 🟢  │ 通知 · 🟢        │
  └────────────────┴──────────────────┘

▼ 技能 · 3 个
  [技能卡片列表, 和现在一样]

▼ 权限 · 4/7 已授权
  [横滚权限标签, 和现在一样]
```

---

## 自适应规则

三种折叠行的行为略有不同：

| 折叠行 | 全部正常时 | 有异常时 |
|--------|----------|---------|
| 通道 | 摘要显示"全部正常" | 自动展开，显示哪个异常 |
| 技能 | 摘要显示"3 个" | 不适用（技能没有异常概念） |
| 权限 | 全部已授权 → **这行消失** | 4/7 → 摘要显示"4/7 已授权" |

**权限全部授权后自动隐藏。** 用户不需要看到"一切正常"的权限列表——这只在有问题时才有意义。

## 权限去哪儿了

权限有两种合理的去处：

**方案 A（推荐）：权限全部授权后自动隐藏。** 有未授权时显示在折叠行里。用户看到"权限 · 4/7"就知道有问题，点开去授权。全部 OK 后消失。

**方案 B：移到设置页。** 权限是应用配置，不属于运行时监控。但用户可能在运行页看到"通知通道异常"然后想知道是不是权限问题——留在运行页有排查价值。

建议 A。权限留在运行页，但只在有问题时出现。
